## 遊戲設計

### 基本規則
- 登入輸入序號（如：001）進入說明頁，確認後才開始計時
- 總時限 **20分鐘**
- 地圖為 Zelda 地下城風格，**9個房間 3×3 配置**，每次只顯示一個 5×5 房間，換房間有滑入動畫

### 房間配置

| 位置 | 房間名稱 |
|---|---|
| 左上 | 局長室 |
| 中上 | 企劃科 |
| 右上 | 電腦機房 |
| 左中 | 行政科 |
| 正中（起始）| 大禮堂 |
| 右中 | 醫政科 |
| 左下 | 檢驗科 |
| 中下 | 疫苗室 |
| 右下 | 疾管科 |

### 玩家操作
- **鍵盤**：方向鍵 或 WASD
- **觸控**：滑動手勢

### 火焰與答題
- 每房間有 **2 個火焰**（🔥），踩上觸發題目
- 答完後火焰熄滅，**20秒後重生**並更換新題
- 題庫共 **40題**，打亂順序循環出題

### HP 系統
- 初始 **10滴血**（❤️ × 10）
- 答錯扣 1 滴血
- 血量歸零 → 死亡畫面，**3秒後復活回大禮堂**，血量全滿

### 答對特效（按「繼續探索」後 350ms 觸發）
- **+1 浮動文字**：金黃色，從火焰位置往上飄
- **爆炸粒子**：橘紅色，從火焰位置向外噴射（12顆，每顆獨立 keyframes）
- **連擊數**：連續答對顯示「🔥 連擊 ×N」，答錯歸零

### 其他機制
- 每完成 **40題一輪**，跳出結算視窗（可選擇繼續或結束）
- 每 **5 / 10 / 15 分鐘**顯示鼓勵訊息
- 遊戲結束顯示完整統計（答題數、正確率、涵蓋題數、死亡次數、探索房間數）

## 題庫結構（共40題）

| 類別 | 題數 | 說明 |
|---|---|---|
| 前測 | 12題 | 實驗前測驗用題 |
| 後測 | 12題 | 實驗後測驗用題 |
| 共通 | 3題 | 前後測共用題 |
| 題庫 | 13題 | 補充練習題 |

Bloom 認知層次分布：L1（記憶）、L2（理解）、L3（應用）、L4（分析）、L5（評鑑）

---

## Google Sheets 資料欄位（共15欄）

| 類別 | 欄位名稱 | 說明 |
|---|---|---|
| 基礎識別 | Timestamp | 台灣時間（Asia/Taipei）|
| | UserName | 登入序號 |
| | SessionID | 本次遊戲唯一識別碼 |
| 研究核心 | QuestionID | 題目編號 |
| | BloomLevel | Bloom 認知層次（L1–L5）|
| | QuestionCategory | 題目類別（前測/後測/共通/題庫）|
| | ResponseTime_ms | 答題反應時間（毫秒）|
| | IsCorrect | 是否答對（true/false）|
| | ChosenOption | 選擇選項（A/B/C/D）|
| | FirstAttemptCorrect | 是否第一次嘗試即答對 |
| | AttemptNum | 本題累計嘗試次數（跨輪累計）|
| 遊戲行為 | RoomName | 答題當下所在房間 |
| | HPAtAnswer | 答題當下剩餘血量 |
| | DeathCount | 累計死亡次數 |
| | TotalPlayTime | 累計遊玩秒數 |

遊戲結束時另送一筆 `QuestionID = SESSION_END`，TotalPlayTime 為實際遊玩秒數（玩滿20分鐘 = 1200）。
