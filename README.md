## 遊戲設計（V16 現況）

### 核心概念

橫向捲軸RPG戰鬥與防災答題結合。主角固定在畫面左側，背景採用實景圖片（Canvas中景.png）水平捲動製造前進感，怪物從右側滑入就位，玩家透過答題驅動戰鬥結果。

### 畫面佈局

- 上方約1/3：HUD資訊列（時間、正確率、已答題數）＋ HP列 ＋ 關卡進度條 ＋ 戰鬥場景（Canvas）
- 下方約2/3：防災答題區（可捲動）
- 答題後解析與繼續按鈕以浮動面板從畫面底部滑出，玩家不需滑動即可看到
- 全比例配置，手機、平板、電腦均適配，canvas尺寸動態讀取

### 戰鬥場景（Canvas 繪製）

**背景（實景圖片水平捲動）**
- 採用外部圖片 Canvas中景.png，依 canvas 高度等比縮放並水平無縫循環鋪排
- 捲動速度：bgOff0 += 0.9（每幀）
- 遭遇怪物後背景停止捲動，等待答題
- 圖片載入前以深藍色填滿作為備用底色

**主角（Q版圓潤小人，Canvas繪製）**
- 大圓頭、圓潤橢圓身體、腮紅、大瞳孔
- 跑步彈跳動畫、蓄力後仰姿勢、受擊×眼＋後退彈回（knockback）
- 帽子依打倒Boss數升階（見升階系統）
- Boss特效依打倒Boss數疊加（見特效系統）

**怪物（Q版圓滾滾，Canvas繪製）**
- 從畫面右側外滑入就位，有待機搖擺動畫
- 普通怪：圓滾滾造型，光暈漸層，小腳踢踢
- Boss：體型放大1.6–2.0倍，旋轉刺冠，持續光圈特效，出現時畫面震動＋BOSS標籤顯示

### 完整戰鬥流程

**怪物就位：**
怪物從右側畫面外滑入定位，就位後自動出題。玩家看浮動解析面板時，下一隻怪物提前滑入就位（不出題），等玩家按「看完了」後才正式開始下一回合。

**答題：**
點選A/B/C/D選項後，畫面底部立即彈出浮動解析面板（從底部滑入），包含答對/答錯大圖示、正確答案與解析說明、繼續按鈕。玩家主動按按鈕後面板滑回，推進下一步。

**答對（一般怪）：**
主角後仰蓄力（chargeTimer）→ 射出水柱拋物線 → 怪物預警抖動 → 水柱命中爆炸消失 → +1浮動文字 → 浮動解析面板彈出

**答錯（一般怪）：**
怪物吸氣膨脹蓄力（windupTimer）→ 射出火球 → 主角受擊後退彈回 → 怪物捨身消失 → 主角扣1血 → 浮動解析面板彈出

**答對（Boss，未達3擊）：**
主角攻擊動畫 → 水柱命中Boss → Boss HP條減少 → Boss仍站著 → 浮動解析面板彈出

**答對（Boss第3擊）：**
Boss爆炸粒子消失 → 大量彩色粒子特效 → 浮動解析面板彈出（按鈕文字：「💥 Boss擊倒！我看完了 →」）→ 升階通知彈出

**答錯（Boss）：**
怪物吸氣蓄力 → 射出火球 → 主角受擊後退 → 主角扣1血 → Boss仍站著不消失 → 浮動解析面板彈出

### 浮動解析面板設計

- 從畫面底部滑入，疊在題目區上方，不需滑動即可閱讀
- 答對：深綠底色＋綠色頂邊框＋綠色外光暈
- 答錯：深紅底色＋紅色頂邊框＋紅色外光暈
- 頂部大圖示（✅ 或 ❌）＋解析文字＋繼續按鈕
- 按下按鈕後面板滑回底部消失，執行下一步邏輯

### 按鈕文字對照

| 情境 | 浮動面板按鈕文字 |
|---|---|
| 一般怪答對 | 📖 看完了，下一關 → |
| 一般怪答錯 | 📖 看完了，下一關 → |
| Boss答對，尚未擊倒 | 📖 看完了，繼續攻擊 Boss → |
| Boss答錯 | 📖 看完了，繼續挑戰 Boss → |
| Boss第3擊擊倒 | 💥 Boss擊倒！我看完了 → |
| 每輪40題答完 | 獨立結算視窗（繼續冒險🔥 / 結束學習） |

### 連擊系統

連續答對顯示「🔥 連擊 ×N」，答錯歸零。連擊達3次以上，水柱變金色、體積變大、命中爆炸範圍更大。

### 關卡與題庫系統

- 每答一題（答對或答錯）推進一關，關卡數累計不重置
- 每10關出現一隻Boss，Boss需答對3題才能擊倒
- Boss答錯：主角扣血，Boss不消失、HP不扣，繼續出新題挑戰
- 題庫共40題，每輪不重複，隨機排序
- 答對答錯都消耗題目配額（每輪40題共享，含Boss回合）
- 答完第40題後跳出每輪結算視窗，不論當時是打小怪或Boss中途
- **Boss打到一半剛好答完第40題**：玩家按「看完了」後跳結算視窗，結算按「繼續冒險」後接著挑戰Boss（題庫已重洗，Boss進度與HP條保留）
- **Boss擊倒剛好是第40題**：先顯示Boss擊倒＋升階通知，600ms後跳結算視窗
- 每輪結算視窗的「繼續冒險」是唯一重洗題庫的時機
- 結算視窗裡按「繼續冒險」後，若Boss還活著則直接繼續Boss回合，不重新spawn怪物

### 主角升階系統

每打倒一個Boss升階一次，共8個等級，奇數次換帽子，偶數次加特效：

| 打倒Boss數 | 變化類型 | 內容 | 稱號 |
|---|---|---|---|
| 0（初始） | 外觀 | 黑色頭髮 | 一般員工 |
| 1 | 換帽 | 黃色安全帽 | 初級防災員 |
| 2 | 加特效 | 6顆金色✦繞主角旋轉 | 初級防災員 |
| 3 | 換帽 | 深色帽＋橘色反光條 | 防災專員 |
| 4 | 加特效 | 藍紫雙光圈脈衝（相位差） | 防災專員 |
| 5 | 換帽 | 紅色消防頭盔＋金徽章 | 消防員 |
| 6 | 加特效 | 5個🔥飄散頭頂周圍 | 消防員 |
| 7 | 換帽 | 深紅金徽消防帽＋金星 | 防災達人 |
| 8以上 | 加特效 | 閃電弧光隨機跳動 | 防災達人 |

每次升階彈出通知視窗，換帽顯示「⬆️ LEVEL UP！」，加特效顯示「✨ 特效解鎖！」。

### HP與死亡系統

- 初始10滴血，以❤️圖示顯示，答錯扣1滴，同時畫面震動
- 血量歸零：出現死亡覆蓋畫面，倒數3秒後滿血繼續
- 復活後從當前狀態繼續，不重置任何進度

### 計時與特殊訊息

- 說明頁按「出發冒險！」後才開始計時，共20分鐘
- 最後60秒計時器變紅閃爍
- 第5、10、15分鐘彈出鼓勵訊息
- 時間到顯示結束畫面

### 結束畫面統計

- 作答次數（含重複輪）
- 答對次數
- 正確率
- 涵蓋題數（不重複）/40
- 死亡次數
- 最高連擊數
- 依正確率顯示不同鼓勵文字（≥90% / ≥70% / ≥50% / 以下）

## 研究數據欄位（Google Sheets，15欄）

每次答題即時送出一筆資料：

| 類別 | 欄位名稱 | 說明 |
|---|---|---|
| 基礎識別 | Timestamp | 台灣時間（Asia/Taipei） |
| | UserName | 玩家輸入序號（如：001） |
| | SessionID | 本次遊戲唯一識別碼（S+時間戳） |
| 研究核心 | QuestionID | 題目序號（對應上表1–40） |
| | BloomLevel | 認知層次（L1–L5） |
| | QuestionCategory | 題目分類（前測/後測/共通/題庫） |
| | ResponseTime_ms | 答題時間（毫秒，從題目出現到選擇） |
| | IsCorrect | 是否答對（true/false） |
| | ChosenOption | 選擇的選項（A/B/C/D） |
| | FirstAttemptCorrect | 是否第一次嘗試即答對（true/false） |
| | AttemptNum | 本題在此session的累計嘗試次數 |
| 遊戲行為 | RoomName | 作答時所在關卡（stage_N） |
| | HPAtAnswer | 作答時的HP值 |
| | DeathCount | 累計死亡次數 |
| | TotalPlayTime | 累計遊玩秒數 |

遊戲結束時另送一筆 `QuestionID = SESSION_END`，記錄最終HP、死亡次數與實際遊玩總秒數（玩滿20分鐘 = 1200）。
